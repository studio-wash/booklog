# 피쳐 계획서

**Plan ID**: PLAN-000003  
**생성일**: 2026-05-13  

## 피쳐 아이디어

새 책 추가 시 **검색을 기본·주 경로**로 두고, 결과에 **썸네일·저자 등**을 보여 준 뒤 선택하면 제목(등)을 채운다. **직접 제목 입력**은 검색이 안 되거나 원할 때만 쓰는 **보조(접힘/하단)**로 둔다.

## 목적

- 서재에 넣는 책 정보를 **검색 기반으로 일관되게** 넣어 입력 오타·누락을 줄인다.
- FR-7 방향(외부 검색으로 메타 채움)을 **UX 중심**으로 끌어올린다.
- 직접 입력은 **예외 경로**로만 노출해 기본 플로우는 단순하게 유지한다.

## 핵심 기능

### 1. 새 책 추가 플로우 재구성

- **Books** 화면 `+` → 바텀시트(또는 동일 패턴의 전면 시트)를 연다.
- **1순위 영역**: 검색어 입력 + **검색 실행** 버튼(또는 입력 후 자동/디바운스는 구현 단계에서 태스크로 결정). 기존처럼 `dart-define`으로 검색을 끄는 모드가 있다면, 기본 배포는 `API_BASE_URL`이 있으므로 검색이 켜진 상태를 전제로 UI를 짠다.
- **결과 목록**: 네이버(프록시) JSON의 `items[]`를 사용한다. 각 행에 최소 다음을 표시한다.
  - **썸네일**: `image` URL — `Image.network`, 로딩·실패 시 placeholder(회색 박스 또는 아이콘).
  - **제목**: HTML 태그 제거 후 1~2줄 말줄임.
  - **부가 한 줄**: `author`, 필요 시 `publisher` 일부(공간·가독성에 맞게).
- **행 탭**: 해당 항목으로 **제목 필드(내부 상태)를 채우고**, 선택 하이라이트 또는 확인 한 줄을 보여 준 뒤 **총 페이지(선택)** 입력으로 이어지거나 같은 시트 하단에 **저장**을 둔다. (총 페이지는 네이버 응답에 없을 수 있으므로 **수동 입력 유지**.)
- **저장**: 기존과 같이 로컬 DB `insertBook`에 제목·총 페이지 전달.

### 2. 직접 입력(보조)

- 시트 **하단**에 `직접 입력` 등 접이식(`ExpansionTile` 또는 `TextButton`으로 펼침)을 둔다.
- 펼치면 **제목**(필수)·**총 페이지**(선택) 필드만 노출. 검색으로 이미 제목이 채워진 경우에는 덮어쓰기 전 확인 또는 필드에 반영만 하면 된다.
- 검색 결과가 0건이어도 시트를 닫지 않고, 안내 문구 + 직접 입력 유도를 같이 둔다.

### 3. 디자인·구현 메모

- **Material 3** + 기존 `Theme`/`AppTheme` 톤에 맞춘다. 리스트는 `ListTile` 또는 `Card` + 고정 높이 썸네일(예: 48~56dp)로 정돈.
- 시트 높이: `DraggableScrollableSheet` 또는 `isScrollControlled` + `maxHeight` 비슷한 제약으로 **키보드·긴 목록**에서 스크롤 가능하게 한다.
- **데이터 계층**: `book_search_api`가 `List<String>`만 반환하지 않고, `title`, `image`, `author`, `publisher`(선택) 등을 담은 **모델 클래스**(예: `BookSearchHit`) 리스트를 반환하도록 확장한다. API 응답 형식은 `knowledge/reference/api/naver-book-search.md` 및 실제 JSON 필드에 맞춘다.

## 사용자 시나리오

1. 사용자가 Books에서 `+`를 누른다.
2. 검색창에 키워드를 넣고 검색한다 → 썸네일·제목·저자가 보이는 목록이 뜬다.
3. 한 권을 탭한다 → 제목이 채워진다(필요 시 총 페이지 입력).
4. 저장한다.
5. 검색이 실패하거나 원하는 책이 없으면 하단에서 직접 입력을 펼쳐 제목을 적고 저장한다.

## 기술적 고려사항

- **api-server**: 현재 `GET /api/books/search`는 네이버 JSON을 그대로 전달하므로, 필드 확장을 위해 **서버 변경은 필수 아님**(Flutter 파싱만 확장).
- **이미지**: 외부 URL이므로 Android/iOS **cleartext/ATS** 이슈 없음(HTTPS 썸네일 전제). `bookthumb.phinf.naver.net` 등은 일반적으로 HTTPS.
- **성능**: 검색 요청은 디바운스(300~500ms) 또는 명시 버튼만 사용해 호출 수를 제한한다.

## MVP 대비 범위

- 본 플랜은 **서재「새 책 추가」UX**에 한정한다. 기록 화면(`log_entry`)의 책 검색 연동은 포함하지 않는다(필요 시 후속 플랜).

## 완료 기준

- 새 책 추가 시 검색·결과 목록(썸네일 포함)·선택 후 저장까지 동작한다.
- 직접 입력 경로로만 제목을 넣어도 저장 가능하다.
- 기존 단위/통합 테스트가 있다면 깨지지 않게 하고, 없으면 수동 시나리오로 위 플로우를 확인한다.
