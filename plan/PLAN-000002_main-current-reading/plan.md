# 피쳐 계획서

**Plan ID**: PLAN-000002  
**생성일**: 2026-05-12  
**관련**: PLAN-000001 하루한장 MVP — 메인(잔디) 화면 확장

## 피쳐 아이디어

메인 잔디 그래프 **아래 빈 공간**에, 사용자가 **마지막으로 기록을 남긴 책**과 그 책에 대한 **누적 읽기 진행**(페이지 정보)을 한눈에 보여준다.

## 목적

- 잔디만 보면 “지금 무엇을 읽고 있는지”가 드러나지 않아 맥락이 끊긴다. 메인에서 **최근 활동 책**을 바로 확인·이어 쓰기 유도한다.
- MVP는 **페이지 수 중심**이므로, “현재 페이지”는 DB에 절대 좌표로 없고 **일별 `pages` 합산**으로만 표현 가능함을 사용자에게 자연스럽게 보여 준다.

## 데이터·표시 정의

- **대상 책**: 전역 기준 `reading_entries` 중 **`created_at`이 가장 큰** 한 행의 `book_id`에 해당하는 책. (같은 시각 중복 시 SQLite 반환 순에 따름 — MVP에서 허용.)
- **누적 읽은 페이지**: 해당 `book_id`에 대해 기존과 동일하게 `SUM(pages)` (`AppDatabase.totalPagesReadForBook`).
- **총 페이지**: `books.total_pages`가 있으면 `누적 / 총` 형식과 **선형 진행률 바**(0~1, 총이 0이면 바 숨김).
- **총 페이지 없음**: `누적만` 표시 (예: `128 pages read`).
- **기록 전(엔트리 없음)**: 카드 영역에 짧은 안내 문구만 표시. 추가 버튼은 넣지 않고 기존 FAB로 기록 유도.

## 핵심 기능

### 1. 메인 UI — 현재 읽기 카드

- 위치: `GrassScreen` 본문에서 **365일 잔디 스트립(`Expanded`) 아래**, `FloatingActionButton`과 겹치지 않도록 하단 패딩 유지.
- 형태: `Card` 또는 `Material` + 둥근 모서리, 제목 한 줄(말줄임), 부제에 누적/총·**마지막 기록일**(해당 엔트리의 `calendar_date` 또는 `created_at` 중 사용자에게 의미 있는 쪽을 짧은 날짜 형식으로).
- 탭: `context.push('/log?bookId=<id>')` 로 기록 화면 진입(이미 라우터 지원).
- `readingDataTickProvider` 갱신 시 카드도 다시 로드되도록 **Riverpod `FutureProvider`**(또는 기존 패턴과 동일한 `autoDispose`)로 묶는다.

### 2. 데이터 계층

- `AppDatabase`에 예: **`Future<ReadingEntry?> latestReadingEntry()`** — `ORDER BY created_at DESC LIMIT 1`.
- 카드용 프로바이더: 위 엔트리로 `bookId` 결정 → `Book` 조회 + `totalPagesReadForBook` 호출. 한 번의 화면 로드를 위해 DB 왕복을 줄이려면 **단일 rawQuery**로 조인·집계하는 메서드 추가도 가능하나, MVP는 **2~3회 쿼리**로도 충분.

### 3. 스펙·문서

- `spec/features/booklog-mvp/booklog-mvp.md`에 **FR-10(또는 기존 FR 번호 체계에 맞는 새 ID)** 한 줄: 메인에 마지막 기록 책·누적 페이지 요약 표시, 탭 시 해당 책으로 기록 화면 진입.
- `plan/PLAN-000001_*` / `tasks.md`는 본 플랜 완료 후 `/tasks PLAN-000002`로 후속 태스크 분해 가능.

## 사용자 시나리오

1. 사용자가 며칠 전 책 A로 기록을 남겼다 → 메인 하단 카드에 **책 A 제목**, **누적 N페이지**, 총 페이지가 있으면 **진행 바**.
2. 카드를 탭하면 **책 A가 선택된 상태**로 기록 화면이 열린다.
3. 아직 어떤 기록도 없으면 카드 영역에 **안내 문구**만 보인다.

## 기술적 고려사항

- **성능**: 메인 진입마다 가벼운 쿼리 1~3회; 인덱스는 이미 `created_at` 기준으로 충분히 빠름.
- **테스트**: 인메모리 DB에 엔트리 삽입 후, 최신 엔트리가 아닌 다른 책이 먼저 있어도 **가장 최근 created_at** 책이 카드에 나오는지 단위 테스트 1건.
- **다국어**: 기존 앱 문자열이 영문이면 카드 라벨도 동일 톤 유지.

## 범위 밖 (이번 플랜에서 제외)

- “여러 권 동시에 읽는 중” 다중 카드, 읽기 목록(pinned) 등.
- 절대 페이지 번호(예: 127쪽까지 읽음) 입력·저장 — 스키마 변경 없음.
